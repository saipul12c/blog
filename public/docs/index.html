Error 429

Jika limit terlampaui, server akan mengembalikan status 429 dengan pesan berikut:

{
  "error": "Rate limit API terlampaui. 
  Maksimal 30 req/menit."
}

Error Handling

Status Meaning Description
200 OK Permintaan berhasil diproses.
404 Not Found Resource yang diminta tidak ditemukan.
429 Too Many Requests Terlalu banyak permintaan dalam waktu singkat.
500 Server Error Terjadi kesalahan pada internal server kami.

Best Practices

Caching Strategy

Mengingat limitasi 30 req/menit, sangat disarankan untuk mengimplementasikan caching di sisi client (SWR, React Query, atau localStorage) untuk data yang jarang berubah seperti daftar kategori atau profil penulis.

Environment Variables

Jangan pernah melakukan hard-code Base URL di dalam kode aplikasi Anda. Gunakan variabel lingkungan (seperti process.env.API_URL) untuk memudahkan perpindahan antar environment (dev/prod).
